主要サービスの比較

🚀 Renderでデプロイ実践（前半）

ステップバイステップガイド

  1. 1Renderアカウント作成

    https://render.com でGitHubアカウントでサインアップ

    GitHubアカウントでログインすると、リポジトリの連携が簡単になります

  2. 2新規Webサービス作成

    Dashboard → New → Web Service → Connect GitHub repository → Select your repository

    リポジトリを選択すると、自動的に設定が検出されます

  3. 3基本設定の入力

    Name: my-app-name（URLになります）
    Environment: Node
    Region: Singapore（日本に近い）
    Branch: main

🚀 Renderでデプロイ実践（後半）

  1. 4ビルド & スタートコマンド設定

    Build Command: npm install
    Start Command: npm start

  2. 5環境変数の設定

    Environment → Add Environment Variable

    DATABASE_URL = [your-database-url] JWT_SECRET = [your-secret-key] NODE_ENV = production

  3. 6デプロイ開始

    ✅ Create Web Serviceをクリックすると自動的にデプロイが開始されます！
    5-10分程度でデプロイが完了し、公開URLが発行されます。
💡 ポイント： デプロイログをリアルタイムで確認できるので、エラーが出た場合はすぐに対処できます。

🌐 カスタムドメインとHTTPS

プロフェッショナルな公開

  1. 1ドメイン取得

    お名前.com、Google Domains、Namecheapなどで取得（年間1,000円〜）

  2. 2DNSレコード設定

    Type: CNAME Name: www Value: your-app.onrender.com Type: A Name: @ Value: [RenderのIPアドレス]

  3. 3Renderでカスタムドメイン追加

    Settings → Custom Domains → Add Custom Domain

  4. 4SSL証明書の自動発行

    ✅ RenderはLet's Encryptを使用して自動的にSSL証明書を発行・更新します
💡 HTTPSは現在のWeb標準です。Renderなら無料で自動設定されます。

🔧 よくあるトラブルと解決法

❌
ビルドエラー

原因： package.jsonの不整合

npm install --save [missing-package] rm package-lock.json npm install
❌
ポートエラー

原因： ポート番号の固定

const PORT = process.env.PORT || 3000;
❌
DB接続失敗

原因： 環境変数の未設定

環境変数DATABASE_URLを確認 SSL設定の追加が必要な場合あり
❌
CORSエラー

原因： オリジン設定ミス

本番URLをCORS設定に追加 credentials: trueの設定確認
Claude Codeを使ったデバッグ：
「Renderのログに以下のエラーが出ています。原因と解決方法を教えてください」
→ エラーログをペーストすると、Claude Codeが解決策を提示します
